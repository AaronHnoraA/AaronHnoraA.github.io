# Aaron's Org Knowledge Base

This directory is the source tree for Aaron's personal knowledge base and published website.

It is maintained primarily as Org notes, and the site is generated from those notes by Emacs. The goal is not to split “note-taking” and “publishing” into separate systems, but to keep one content tree that supports thinking, writing, research, and publishing together.

## What This Directory Is

`~/HC/Org/` serves three roles at the same time:

- a day-to-day note and capture space
- a long-lived personal knowledge base
- the content source for a published website

This is intentionally not a generic Org template repo. It is a personal system that keeps evolving with use.

## Working Model

The directory is organized around a small number of stable zones:

- `daily/`
  Fast capture and in-progress notes.
- `roam/`
  Evergreen notes and longer-lived knowledge.
- `attachments/`
  Raw files such as PDFs, images, scans, and other source material.
- `references/`
  Citation and bibliography material.
- `diary.org`
  A linear private journal.
- `public/`
  Generated site output.

The basic idea is simple:

- write quickly into `daily/`
- refine durable material into `roam/`
- keep raw files in `attachments/`
- publish from the Org source tree

## Publishing

The website is generated by Emacs from the Org source files.

In this setup, the publishing entry points live in:

- `publish.el`
- `Makefile`

The current pipeline uses Emacs and `org-publish` to render the site into `public/`.

## Relationship to the Emacs Config

The authoring environment for this directory lives in the Emacs configuration that powers it: capture, org-roam, LaTeX preview, citations, Jupyter support, and related workflows are maintained there.

Operational guidance such as tag conventions, knowledge-base structure, and Org workflow notes is documented with that Emacs configuration rather than kept in this README.

## Scope

This directory is meant to be:

- a personal knowledge base
- a writing and research workspace
- a website generated from Org content

It is not trying to be a reusable CMS, a general-purpose static site starter, or a universal note-taking template.
